//
// Порядок создания проекта с использованием google_test:
//

Для начала имеем проект ('NameProgect') с классами (или фунциями, отдельные файлы с функциями), которые нужно протестировать.

1) Добавляем проект с файлали googletest:
1.1) В имеющийся проект добавляем статический проект Win32, с отключенным авто. кампиляцией заголовка ( щелчёк ПКМ по "Решение'NameProgect'" -> "Добавить" -> "Создать проект.." -> "Проект Win32" (не забыть ввести имя проекта) -> "ОК" -> "Далее" -> Выбрать "Статическая библиотека" и убрать галку с "Предварительно скомпилированный заголовок" -> "Готово" )
1.2) Зайти в свойства проекта ( созданного как "Статическая библиотека"(NameStaticProgect) ) и в "Свойства конфигурации"->"Каталоги VC++" -> "Каталоги включения" : Добавить пути к папке с googletest ( ...\googletest-master\googletest или в корне MS VS2012 - C:\Program Files\visual studio 2012\VC\googletest ) и к заголовкам файлов googletest ( ...\googletest-master\googletest\include или в корне MS VS2012 - C:\Program Files\visual studio 2012\VC\googletest\include )
1.2.1) В версиях MS VC2012 и ранее в свойствах препроцессора ( "Свойства конфигурации" -> "C/C++" -> "Препроцессор" -> "Определения препроцессора" ) добавить параметр - '_VARIADIC_MAX=10'
1.3) Добавляем файлы ( "gtest-all.cc", "gtest_main.cc" ) из папки с googletest к ( созданного как "Статическая библиотека" ) проекту ( щелчёк ПКМ по "NameStaticProgect" -> "Добавить" -> "Существующий элемент.." -> Добавляем файлы из папки "...\googletest-master\googletest\src" -> "Добавить"

2) Добавляем проект с тестами, которые будут тестировать классы ( или методы ) проекта 'NameProgect':
2.1) Создаём консольный проект ( щелчёк ПКМ по "Решение'NameProgect'" -> "Добавить" -> "Создать проект.." -> "Консольное приложение Win32" (не забыть ввести имя проекта) -> "ОК" -> "Далее" -> Выбрать "Консольное приложение" и убрать галку с "Предварительно скомпилированный заголовок" -> "Готово" )
2.2) Зайти в свойства проекта ( созданного как "Консольное приложение Win32" ) и в "Свойства конфигурации"->"Каталоги VC++" -> "Каталоги включения" : Добавить пути к папке с googletest ( ...\googletest-master\googletest или в корне MS VS2012 - C:\Program Files\visual studio 2012\VC\googletest ), к загокам файлов googletest ( ...\googletest-master\googletest\include или в корне MS VS2012 - C:\Program Files\visual studio 2012\VC\googletest\include ) и путь к папке исходного ( проекта, который нужно тестировать ) проекта ( "..\..\'NameProgect'\'NameProgect'" )
2.2.1) В версиях MS VC2012 и ранее в свойствах препроцессора ( проекта созданного как "Консольное приложение Win32" ) ( "Свойства конфигурации" -> "C/C++" -> "Препроцессор" -> "Определения препроцессора" ) добавить параметр - '_VARIADIC_MAX=10'
2.3) Зайти в свойства проекта ( созданного как "Консольное приложение Win32" ) во вкладке "Общие свойства" добавить ссылки на проекты с googletest и исходного проекта ( 'NameProgect' ) : -> "Добавить новую ссылку" и установить галки на против названий этих проектов ( "NameStaticProgect" и 'проекта созданного как "Консольное приложение Win32"' )
2.4) Удалить ненужные файлы ( "targetver.h", "stdafx.h", "stdafx.cpp" ), так как они не понадобятя. Так же удаляем всё в файле с функцией _tmain()

3) Проект для написания тестов готов, осталось только:
3.1) Подключить необходимые файлы с googletest и файлы с тестиуемыми классами ( или методали ) ( подключения: #include "gtest\gtest.h" и #include "'NameFaileForTesting'" )
3.2) Написать сами тесты
